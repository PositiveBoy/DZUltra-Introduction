### 1.1 全局规则

- 页面使用 `h-screen` 全屏，去掉 `max-w-7xl` 限制
- 左右面板通过可拖拽分隔条调整占比，默认 MobileShell 35% : 右侧 65%
- MobileShell 最小宽度 320px，最大 50%
- 拖拽使用 `react-resizable-panels` 库
- 所有模块一屏可见，模块内部可上下滚动

## 2. 顶部项目介绍区

### 2.1 折叠行为

## 3. 精简状态条

### 3.1 位置与行为

- 位于右侧面板顶部，始终可见，不随内容滚动
- 高度固定 48px
- 背景色与右侧面板一致，底部 1px 分隔线

### 3.2 内容

```
[●Router ✓] → [●Discovery ✓] → [●Preference ✓] → [●Grounding ⟳] → [●Solver ○] → [●Evaluator ○] → [●Explanation ○]  │  amap🟢 qw🟢 mt🟡mock  │  4.2s/10s
```

三段式布局：
- **左段**：Agent 进度条（7 个圆点 + 名称 + 箭头）
- **中段**：Provider 状态灯
- **右段**：耗时进度条

### 3.3 Agent 进度

- ✓ 完成：绿色圆点 + 绿色文字
- ⟳ 进行中：蓝色脉冲动画圆点 + 蓝色文字
- ○ 待执行：灰色圆点 + 灰色文字
- ✗ 失败：红色圆点 + 红色文字

点击某个 Agent 圆点 → 右侧 Debug Trace 跳到对应事件并高亮

### 3.4 Provider 状态灯

- 🟢 真实接入：绿色圆点 + Provider 缩写
- 🟡 Mock 降级：黄色圆点 + "mock" 标签
- 🔴 超时不可用：红色圆点 + "timeout" 标签

Provider 缩写：amap / caiyun / mock

### 3.5 耗时进度条

- 显示当前总耗时 / 10s 预算
- 进度条颜色：<7s 绿色，7-9s 黄色，>9s 红色
- 无活跃 Trace 时显示 "—"

### 3.6 数据来源

- Agent 进度：从 `activeTrace.events` 提取当前执行到的 Agent
- Provider 状态：从 Trace 事件的 `tool_output.provider` / `reliability` 字段提取
- 耗时：从 `activeTrace.events` 累加 `duration_ms`

## 5. 详情内容区 — 三个 Tab

### 5.1 Tab 切换

三个 Tab：**Debug Trace** | **History** | **Mock 数据**

- Debug Trace 为默认选中
- Tab 切换不丢失状态（已展开的详情、滚动位置等）
- Tab 栏高度 40px，与精简状态条之间无间距

### 5.2 Debug Trace Tab

#### 5.2.1 Provider 概览卡片

- 位于 Tab 内容区顶部，可折叠
- 折叠后只显示一行状态灯
- 展开后显示：
  - 各 Provider 连接状态（已接入 / Mock 降级 / 超时）
  - 最近降级记录（如"caiyun 天气 API 超时(3.1s) → Mock"）
  - 配额使用情况（如有）

#### 5.2.2 Agent 事件时间线

左侧列表 + 右侧详情，类似现有实现但精简：

**左侧列表**（精简模式）：
- 每个 Agent 一行：圆点 + 名称 + 耗时 + 状态
- 点击展开右侧详情

**右侧详情**（点击展开）：
- 输入 / 处理 / 输出三栏
- Tool 调用列表（含 provider 名称、参数、返回摘要、reliability）
- 约束账本变化（该步骤发现/修改了哪些约束）
- LLM span（模型名、token 用量、耗时、成本）— V3 真实数据，V2 模拟
- 流式展示：LLM 输出逐字出现，Provider 调用步骤级更新

#### 5.2.3 方案排序精简卡片

- 位于事件时间线下方
- 3 个方案的精简对比：标签 + 匹配度条
- 点击展开评分拆解

### 5.3 History Tab

- 历史 Run 列表（从 `/traces` API 加载）
- 点击某条 → 回放该 Run 的完整 Trace
- 不做双 Run 并排对比

### 5.4 Mock 数据 Tab

#### 5.4.1 生成器区域

- 输入表单：城市、商圈、用户类型、游玩主题
- 调用 `/mock/generate-user`、`/mock/generate-pois` API
- 生成结果先预览，再决定是否用于当前演示 Run
- 展示校验结果、地图坐标来源和 fallback 状态

#### 5.4.2 图片管理

- 已上传的 POI 图片、推荐菜图片列表
- 上传一次后复用（持久化到本地）
- 图片缩略图 + POI 名称 + 上传时间

#### 5.4.3 当前 Mock 数据预览

- 用户档案（ID、名称、场景、偏好标签）
- POI 列表（名称、区域、分类、评分、排队时间）
- UGC 摘要

#### 5.4.4 Fallback 配置

- 哪些接口优先真实
- 超时阈值设置
- 降级策略选择

## 6. 左右同步

### 6.1 MobileShell → 右侧面板

- MobileShell 进入某个 Agent 步骤时，精简状态条对应圆点变为 ⟳
- MobileShell 完成某个步骤时，对应圆点变为 ✓
- MobileShell 方案切换时，Debug Trace 展示该方案的匹配度分和排序理由
- MobileShell 微调时，Debug Trace 追加微调事件

### 6.2 右侧面板 → MobileShell

- 点击精简状态条的某个 Agent 圆点，MobileShell 滚动到对应内容块
- 点击 Debug Trace 的某个事件，MobileShell 高亮对应步骤

### 6.3 同步数据来源

- 所有同步基于 Zustand store 的 `activeTrace` 和 `mobileView`
- 新增 `activeAgentStep` 字段到 store，表示当前执行到的 Agent
- 新增 `providerStatuses` 字段到 store，表示各 Provider 的连接状态
