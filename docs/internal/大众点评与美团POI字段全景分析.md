# 大众点评 & 美团 POI 字段全景分析

> 使用方式：这是 POI 字段和 Mock 数据设计的参考资料，不是每个开发任务都要读取的主规范。后续 Agent 默认先读 `TECH_AND_INTERACTION_PLAN.md`，再按任务读取 `FRONTEND_SPEC.md`、`BACKEND_SPEC.md` 或 `AGENT_STRATEGY.md`；只有在设计 POI 字段、榜单标签、交易入口或 UGC 摘要时，再打开本文档。

> 为美团行程规划 Agent 提供数据基础，梳理两大平台商户 POI 的全部可用信息维度。
> 生成时间：2026-05-21 | 置信度：4/5 | 时效性：2026Q1

---

## 一、平台概览

| 维度 | 大众点评 | 美团 |
|------|---------|------|
| 定位 | 本地生活信息与评价平台 | 本地生活交易与服务平台 |
| 核心能力 | 评价体系、榜单体系、内容种草 | 交易闭环（团购/外卖/预订）、到店服务 |
| POI覆盖 | 餐饮、景点、酒店、丽人、休闲、亲子、教育、医疗、家装等 | 餐饮、外卖、酒店、景点、电影、丽人、休闲、医美等 |
| 数据来源 | 官方POI开放API + 前端页面 + 用户UGC | 开放平台API + 前端页面 + 商户后台 |

## 二、大众点评 POI 字段体系

### 2.1 基础标识信息

| 字段 | 类型 | 说明 | Agent决策价值 |
|------|------|------|-------------|
| `shopId` / `openshopid` | string | 商户唯一ID | ⭐⭐⭐ 唯一标识，跨系统关联 |
| `name` | string | 商户名称 | ⭐⭐⭐ 展示、搜索匹配 |
| `branch_name` | string | 分店名 | ⭐⭐ 区分连锁门店 |
| `brandName` | string | 品牌名 | ⭐⭐⭐ 品牌偏好匹配 |
| `openstatus` | int | 商户状态(0:下线 1:在线) | ⭐⭐⭐ 过滤无效商户 |
| `highquality` | int | 高质量标志(1:高质 0:非高质) | ⭐⭐ 质量初筛 |
| `isOverseas` | bool | 是否海外商户 | ⭐ 境外行程区分 |

### 2.2 地理位置信息

| 字段 | 类型 | 说明 | Agent决策价值 |
|------|------|------|-------------|
| `address` | string | 商户地址 | ⭐⭐⭐ 导航、行程串联 |
| `city` | string | 所在城市 | ⭐⭐⭐ 城市级筛选 |
| `latitude` | float | 纬度 | ⭐⭐⭐ 路径规划、距离计算 |
| `longitude` | float | 经度 | ⭐⭐⭐ 路径规划、距离计算 |
| `mallInfo` | object | 商场信息（含热门商户、美食排行等） | ⭐⭐ 商场内行程规划 |
| 商圈 | string | 所属商圈 | ⭐⭐⭐ 商圈级行程规划 |

### 2.3 分类信息

| 字段 | 类型 | 说明 | Agent决策价值 |
|------|------|------|-------------|
| `categories` | list | 商户所属类别（叶子类目） | ⭐⭐⭐ 品类筛选 |
| 大类 | string | 如：美食、景点/周边游、酒店/民宿、丽人、休闲/玩乐 | ⭐⭐⭐ 一级分类 |
| 中类 | string | 如：火锅、本帮江浙菜、水上项目 | ⭐⭐⭐ 二级分类 |
| 小类 | string | 如：重庆火锅、老北京火锅 | ⭐⭐ 精细分类 |
| `special` | list | 商户特色服务标签 | ⭐⭐⭐ 偏好匹配（如"有包间""可停车"） |

### 2.4 评分体系（核心决策字段）

| 字段 | 类型 | 说明 | Agent决策价值 |
|------|------|------|-------------|
| `star` | float | 星级评分（0-5星，如3.5、4.0、4.5） | ⭐⭐⭐⭐⭐ 最核心的质量指标 |
| 口味评分 | float | 口味维度评分 | ⭐⭐⭐⭐ 餐饮类核心指标 |
| 环境评分 | float | 环境维度评分 | ⭐⭐⭐ 约会/商务场景关键 |
| 服务评分 | float | 服务维度评分 | ⭐⭐⭐ 服务敏感型用户关键 |
| 食材评分 | float | 食材维度评分（部分类目） | ⭐⭐ 高端餐饮参考 |
| `reviewCount` | int | 评论总数 | ⭐⭐⭐ 评分可信度参考 |
| `reviewTags` | list | 评论标签（tag + hit数） | ⭐⭐⭐ 快速了解商户特征 |

### 2.5 价格信息

| 字段 | 类型 | 说明 | Agent决策价值 |
|------|------|------|-------------|
| `avgprice` | int | 人均价格 | ⭐⭐⭐⭐⭐ 预算匹配核心字段 |
| `dealInfo` | list | 团购/优惠信息（名称、原价、折扣价、图片） | ⭐⭐⭐ 性价比决策 |

### 2.6 营业信息

| 字段 | 类型 | 说明 | Agent决策价值 |
|------|------|------|-------------|
| `business_hour` | string | 营业时间 | ⭐⭐⭐⭐ 行程时间规划核心 |
| `telephone` | string | 电话号码 | ⭐⭐ 预订确认 |
| 营业状态 | string | 营业中/休息中/已歇业 | ⭐⭐⭐⭐ 实时可用性判断 |

## 六、Agent 数据获取建议

### 6.1 推荐数据源组合

```
行程规划 Agent = 大众点评(评价+榜单+内容) + 美团(交易+价格+实时状态)
```

### 6.2 数据获取路径

| 路径 | 可获取字段 | 限制 |
|------|----------|------|
| 大众点评POI开放API | 基础信息+评分+推荐菜+评论 | 需申请，字段有限 |
| 大众点评前端页面 | 全部字段（含榜单标签） | 反爬严格 |
| 美团开放平台API | 门店基础+交易+配送 | 需商户授权 |
| 美团前端页面 | 列表+详情+团购 | 反爬严格 |
| CnOpenData等数据集 | 历史快照数据 | 非实时 |

### 6.3 行程规划场景字段映射

| 场景 | 关键字段 | 决策逻辑 |
|------|---------|---------|
| 早餐安排 | 营业时间+分类(早餐)+人均+距离 | 6-9点营业的餐饮，距离酒店近 |
| 午餐推荐 | 榜单(必吃榜)+口味评分+人均+推荐菜 | 必吃榜优先，口味>4.5，预算内 |
| 下午茶 | 分类(咖啡/甜品)+环境评分+距离 | 环境>4.0，距离景点近 |
